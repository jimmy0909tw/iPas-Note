一、機器學習基礎概念與演算法

1. 強化學習 (Reinforcement Learning, RL)
PPO (Proximal Policy Optimization)
定義：基於策略梯度的強化學習演算法，解決傳統策略梯度不穩定問題。
  • 核心思想：通過「裁剪」機制限制每次策略更新的幅度，提高穩定性。
  • 優點：訓練穩定性高、數據效率較高、實現相對簡單、性能優越。
  • 適用任務：複雜、連續/高維動作空間的控制任務（機器人、遊戲 AI）、需要穩定訓練的場景。
  • 與 RLHF 的關係：PPO 是 RLHF 中優化大型語言模型 (LLM) 以對齊人類偏好的核心演算法。
RLHF (Reinforcement Learning from Human Feedback)
定義：基於人類回饋的強化學習，旨在讓 LLM 更符合人類偏好。
  • 核心流程：預訓練 LLM -> 監督式微調 -> 訓練獎勵模型 (用人類回饋) -> 使用 RL (通常是 PPO) 優化 LLM 以最大化獎勵。
  • 主要策略目標：最大化累計獎勵 (這是 RL 的根本目標)。
2. 機器學習任務類型
迴歸 (Regression)
  • 目標：預測連續的數值型輸出。
  • 輸出範例：房價、溫度、銷售額。
  • 常用演算法：線性迴歸、決策樹迴歸、隨機森林迴歸。
  • 常用評估指標：MSE, RMSE, MAE, R²。
分類 (Classification)
  • 目標：預測離散的類別標籤。
  • 輸出範例：是/否、疾病類型、垃圾郵件/非垃圾郵件。
  • 常用演算法：邏輯迴歸、決策樹分類、SVM、K-近鄰。
  • 常用評估指標：準確度、精確度、召回率、F1 分數、混淆矩陣、ROC/AUC。
3. 特定機器學習演算法
邏輯迴歸 (Logistic Regression)
  • 類型：主要用於分類任務（特別是二元分類）。
  • 原理：將線性組合的分數通過 Sigmoid 函數轉換為 0 到 1 之間的機率值，然後根據閾值進行分類。
  • 白話比喻：像一個「機率判官」，根據特徵計算「潛力分數」，再用 S 型曲線轉為「上場機率」，最後設「門檻」做二選一判斷。
  • 優點：解釋性好、計算效率高、輸出機率、易於實現。
  • 缺點：線性決策邊界、對異常值敏感。
隨機森林 (Random Forest)
  • 類型：屬於集成學習 (Ensemble Learning) 中的一種。
  • 原理：訓練多個「弱學習器」（決策樹），並將它們的預測結果組合起來（投票/平均），以獲得更強大、更穩定、泛化能力更好的模型。

二、數據處理與預處理

1. OSEMN 數據處理流程
  • O - Obtain (獲取數據)：識別並收集數據。
  • S - Scrub (清洗數據)：處理缺失值、異常值、錯誤數據、去重等。
  • E - Explore (探索數據)：統計摘要、數據視覺化、模式識別 (EDA)。
  • M - Model (建立模型)：特徵工程、模型選擇、訓練、評估 (Evaluate)。
  • N - Narrate (解釋與展示/傳達結果)：解釋模型結果、數據故事講述、推薦與建議。
  • N (iNfer/Inference)：將訓練好的模型應用到新數據上進行預測/推斷。
2. 資料轉換 (Data Transformation)
特徵標準化 (Standardization)
目的：將數據轉換為均值為 0、標準差為 1 的分佈。 (x′=σx−μ​)
  • 作用：消除量綱影響、加速模型收斂、提升依賴距離/梯度的模型性能。
  • 區別於：特徵縮放 (Normalization/Min-Max Scaling) 是將數據壓縮到固定範圍（如 [0,1]）。
3. ETL (Extract, Transform, Load)
定義：數據整合的過程，用於將數據從多個來源提取、轉換、然後載入到數據倉庫/湖。
  • Extract (擷取)：從異構數據源（資料庫、文件、API 等）中提取原始數據。
  • Transform (轉換)：最複雜的步驟，包括數據清洗（缺失值、異常值、錯誤、去重）、數據格式統一、數據聚合、創建新特徵、數據標準化等。
  • Load (載入)：將轉換後的數據載入到目標系統（數據倉庫、數據湖），可分全量或增量載入。
  • 重要性：確保分析數據的乾淨、一致、可靠。

三、AI 模型評估與改進

1. AI 模型的泛化能力 (Generalization Ability)
定義：模型在未見過的新數據上表現好壞的能力。
關鍵概念：
  • 過擬合 (Overfitting)：模型在訓練數據上表現好，但在新數據上差（死記硬背）。
  • 欠擬合 (Underfitting)：模型在訓練數據和新數據上都表現差（沒學好）。
  • 提升方法：增加數據量、數據增強、特徵工程、正則化、早停法、交叉驗證、集成學習、簡化模型複雜度、Dropout。
2. 正則化 (Regularization)
目的：防止模型過擬合，限制模型參數大小。
L1 正則化 (Lasso Regression)：
  • 懲罰項：權重絕對值的和 (λ∑∣βi​∣)。
  • 效果：將一些權重完全壓縮為零，實現特徵選擇，產生更稀疏的模型。
  • 幾何：菱形等高線，最佳解易落於頂點。
L2 正則化 (Ridge Regression)：
  • 懲罰項：權重平方和 (λ∑βi²​)。
  • 效果：將所有權重壓縮到接近零，但很少完全為零，不具備特徵選擇能力。
  • 幾何：圓形等高線。
3. 圖像/文本生成模型評估指標
BLEU (Bilingual Evaluation Understudy)
  • 用於：文本生成（如機器翻譯、文本摘要），衡量生成文本與參考文本的 N-gram 重疊度。
  • 特點：分數越高越好。
FID (Fréchet Inception Distance)
  • 用於：圖像生成（如 GANs、Diffusion Models），衡量生成圖像與真實圖像特徵分佈的距離。
  • 特點：分數越低越好，反映真實性和多樣性。
Perplexity (困惑度)
  • 用於：語言模型，衡量模型預測下一個詞元的「不確定性」。
  • 特點：困惑度越低越好。
ROUGE (Recall-Oriented Understudy for Gisting Evaluation)
  • 用於：文本生成（如文本摘要），衡量生成文本與參考文本的重疊度，更側重召回率。

四、AI 模型架構與技術

1. 鑑別式 AI (Discriminative AI) / 判別式模型
  • 原理：學習不同類別資料之間的「決策邊界」，直接學習 P(Y∣X) (條件機率)。
  • 目標：區分不同類別的數據。
  • 常見模型：邏輯迴歸、SVM、決策樹、隨機森林、大多數深度學習分類模型。
2. 生成式模型 (Generative Model)
VAE (Variational Autoencoder - 變分自動編碼器)
  • 類型：生成式模型。
  • 原理：學習數據的潛在表示 (Latent Representation)，並確保潛在空間連續且有意義。
  • 結構：編碼器 (Encoder) 輸出潛在分佈的均值和方差；解碼器 (Decoder) 從潛在分佈中採樣並生成數據。
  • 目標函數：重建損失 (Reconstruction Loss) + KL 散度損失 (KL Divergence Loss)。
  • 比喻：畫家學習「創意密碼」的「創意範圍」，再從中隨機選取生成新畫作。
編碼器-解碼器架構中的詞元選擇
貪婪搜索 (Greedy Search)：
  • 最簡單的方法：每次都選擇機率最高的詞元作為輸出。
  • 優點：計算效率最高。
  • 缺點：可能陷入局部最優解。
集束搜索 (Beam Search)：
  • 改進方法：同時追蹤 k 條機率最高的詞元序列。
  • 優點：比貪婪搜索效果好，平衡效率與性能。
3. Position Encoding (位置編碼)
目的：將序列中詞元的順序或位置資訊注入模型輸入，解決 Transformer 等模型缺乏序列順序感知能力的問題。
類型：
  • 絕對位置編碼：為每個位置生成固定或可學習的向量。
    • Sinusoidal (正弦)：固定計算，無需學習，能處理任意長序列。
    • Learned (可學習)：作為參數在訓練中學習，靈活但無法外推。
  • 相對位置編碼：在注意力計算中納入詞元對的相對距離資訊。
  • 重要性：Transformer 模型理解語義、語法和句子結構的關鍵。

五、數據與技術基礎

1. 高維度數據 (High-Dimensional Data)
定義：數據集中包含大量特徵或變數的數據。
特徵：
  • 維度詛咒 (Curse of Dimensionality)：數據在高維空間中極度稀疏，導致距離難以區分，模型易過擬合。
  • 數據稀疏性、計算複雜度高、特徵冗餘/噪音、視覺化困難。
  • 處理方法：特徵選擇、特徵提取/降維 (PCA, t-SNE)、選擇適合演算法。
2. 大數據的 5V 特性
  • Volume (資料量/海量)：數據規模巨大。
  • Velocity (處理速度/高速)：數據產生和處理速度快。
  • Variety (資料多樣性/多樣)：數據類型和格式多樣化（結構化、半結構化、非結構化）。
  • Veracity (資料真實性/真實)：數據的質量、準確性、可信度。
  • Value (資料價值/價值)：從數據中提取有意義的洞察和商業價值。

六、AI 法規與雲端服務

1. 歐盟《人工智慧法案》(EU AI Act)
核心原則：基於風險的分級管理。
風險分類：
  • 不可接受的風險 (Unacceptable Risk)：完全禁止，如社會信用評分、實時遠端生物識別識別（公共場所執法目的）。
  • 高風險 (High-Risk)：嚴格監管，如醫療設備中 AI、用於招聘的 AI、一般生物識別系統。
  • 有限風險 (Limited Risk)：透明度義務，如聊天機器人、Deepfakes。
  • 最低風險 (Minimal Risk)：鼓勵自願行為準則。
大規模生物識別系統的風險分類：
  • 如果是公共場所用於執法目的的「實時遠端生物識別識別系統」，則屬於不可接受的風險。
  • 一般性的「生物識別系統」（即使規模大但用途不同）則屬於高風險。
2. Azure OpenAI 服務 與 Azure AI 服務
Azure AI 服務
  • 定位：更廣泛、更通用、開箱即用的 AI 服務集合。
  • 功能：提供預建置的模型和 API，用於常見的 AI 任務（視覺、語音、語言、決策等），多為辨別式 AI。
Azure OpenAI 服務
  • 定位：專注於提供 OpenAI 最先進的生成式 AI 模型（GPT-3.5/4, DALL-E, Embeddings, Whisper）。
  • 優勢：結合 Azure 企業級安全、合規性
